VÍDEO DA ENTREGA (OBRIGATÓRIO)
Tempo recomendado: 5 a 10 minutos
Formato: gravação da tela (OBS, celular filmando tela com excelente qualidade, Loom, Xbox
Game Bar, etc.)
Áudio: claro, sem ruídos, você explicando tudo com calma
Resolução mínima: 1080p
O vídeo deve obrigatoriamente mostrar:
Apresentação inicial (30–45s)
● Nome(s) do(s) aluno(s)
● Nome do projeto
● Problema que ele resolve em 2 frases
● Contextualização rápida do que será mostrado
Exemplo:
“Olá, meu nome é ___, este é o projeto SmartQueue, que automatiza filas de
atendimento. etc….”
Arquitetura do sistema (1 min)
Mostrar um diagrama simples:
● microserviços
● banco(s) de dados
● RabbitMQ
● front-end
● como eles conversam
Você pode mostrar um desenho no Miro, Excalidraw, Figma ou até papel — desde que seja
bonito e legível.
Demonstração do Docker (1 min)
Mostre:
● docker-compose up
● containers subindo
● mensagens do Rabbit
● serviços disponíveis
Demonstração das Funcionalidades (3–4 min)
Mostrar TUDO funcionando, como se fosse vender o produto:
● Criação de dados (POST)
● Listagem (GET)
● Edição (PUT)
● Exclusão (DELETE)
● Parte gráfica funcionando
● Ação que dispara evento no RabbitMQ
● Como outro serviço reage ao evento
Obs.:
Se usar Angular, mostre a UI atualizando automaticamente ou via refresh.
Demonstração da Mensageria (RabbitMQ) (40s)
Precisa aparecer no vídeo:
● mensagem indo para a fila
● consumer recebendo
● efeitos dessa mensagem (atualização de status, logs, etc.)
Por que esse projeto valeria investimento? (1–2 min)
IMPORTANTE: discurso convincente.
Eles devem explicar:
● qual problema real o sistema resolve
● potencial de mercado
● diferenciais
● por que uma empresa investiria
● escalabilidade futura
● tecnologias modernas utilizadas
Exemplo:
“O SmartQueue reduz custos com atendimento presencial, evita filas e melhora a
experiência do usuário. É escalável, moderno, usa microserviços, e poderia
facilmente ser aplicado em clínicas, restaurantes e serviços públicos.”
Encerramento (15–30s)
Algo como:
“Obrigado por assistir. O repositório está no GitHub e o projeto pode ser rodado via Docker
Compose.”
